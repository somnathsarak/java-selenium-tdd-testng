# Test Execution and Extent Report Guide

## Overview
This document provides complete instructions for executing the Java Selenium TDD TestNG framework test suite and locating the generated Extent Reports.

## Test Execution Methods

### Method 1: Using Bash Script (Recommended)

```bash
./run_tests.sh
```

**What it does:**
- Verifies Maven installation
- Cleans and compiles the project
- Executes the TestNG test suite
- Generates test reports
- Displays report location

### Method 2: Using Maven Command

```bash
mvn clean test -Dsurefire.suiteXmlFiles=testng.xml
```

**Parameters:**
- `clean` - Cleans previous build artifacts
- `test` - Compiles and runs tests
- `Dsurefire.suiteXmlFiles=testng.xml` - Specifies TestNG XML configuration

### Method 3: IDE Execution (Eclipse/IntelliJ)

1. Right-click on `testng.xml` file
2. Select "Run As" â†’ "TestNG Suite"
3. View test execution in IDE console
4. Reports generated automatically

## Extent Report Locations

### Primary Report Location
```
<project-root>/test-reports/ExtentReport_YYYY-MM-DD_HH-mm-ss.html
```

**Format:**
- Directory: `test-reports/`
- Filename Pattern: `ExtentReport_<timestamp>.html`
- Timestamp Format: `yyyy-MM-dd_HH-mm-ss`

**Maven Surefire Reports:**

## Accessing Reports

### Step-by-Step Guide

1. **Execute Tests:**
   ```bash
   ./run_tests.sh
   ```

2. **Wait for Completion:**
   - Script displays "SUCCESS: All tests executed successfully!"
   - Provides report location path

3. **Open Report:**
   - Navigate to the provided path
   - Example: `test-reports/ExtentReport_2025-12-24_06-30-15.html`
   - Open in web browser (Chrome, Firefox, Safari, Edge)

4. **Analyze Results:**
   - Review test summary
   - Examine failed tests (if any)
   - Check log details
   - Export report if needed

## Report Analysis

### Success Criteria

For all tests to PASS:
- Application URL must be accessible
- All page elements must be locatable
- Wait conditions must be satisfied
- Assertions must pass

### Common Report Issues

**Issue: NoSuchElementException**
- Element locator has changed
- Page hasn't fully loaded
- Element visibility issues

**Issue: TimeoutException**
- Element not found within wait duration
- Network latency
- Server response delay

**Issue: StaleElementReferenceException**
- DOM has changed since element was located
- Page refresh occurred
- Element was removed from DOM

## Troubleshooting

### Maven Not Found
```bash
# Install Maven or set MAVEN_HOME
export PATH=$PATH:/path/to/maven/bin
```

### Compilation Errors
```bash
# Verify Java version
java -version

# Clear Maven cache
mvn clean dependency:resolve
```

### Report Not Generated
- Verify `test-reports/` directory exists
- Check write permissions
- Verify TestListener is registered in testng.xml
- Review test execution logs for errors
